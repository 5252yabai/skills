# Slack 스레드 게시

## Overview

구조화된 글을 Slack에 올릴 때의 기본 패턴: **요약 메시지로 새 스레드 생성 → 상세 내용을 댓글로 작성**. 톱 메시지는 채널에서 한눈에 읽힐 수 있는 요약이고, 댓글에 전체 내용을 담아 채널 노이즈를 줄인다.

## When to Use

트리거 예시:

- "이 회의록 슬랙에 올려줘"
- "주간 공유 채널에 게시해줘"
- "이 내용 Slack에 공유해줘"
- "/posting-to-slack-thread"

**When NOT to use:**

- 봇이 만든 기존 스레드에 댓글만 달 때 → `daily-scrum-report` 등 전용 스킬 사용
- 단순 1회성 메시지(스레드 불필요) → `slack_send_message` 직접 호출

## Workflow

### Step 1 — 채널 결정

채널이 인자로 주어진 경우 그대로 사용.

채널이 없으면:

```
mcp__claude_ai_Slack__slack_search_channels 로 접근 가능한 채널 목록 조회
→ 채널명/목적을 사용자에게 보여줌
→ 사용자가 선택한 채널의 channel_id 사용
```

### Step 2 — 콘텐츠 준비

**톱 메시지 (요약):**

- **기본 포맷은 불릿 포인트** (사람별/주제별 한 줄). 산문/문단형은 사용자가 명시할 때만
- 본문 불릿은 간결체로 작성 (구어체 X)
- 마지막 줄은 **구어체 한 줄**: `상세 내용은 댓글에 작성할게요.` (격식체 "작성합니다" 사용 금지)
- 필요 시 멘션 포함 (Step 3 참조)
- **이모지 사용 금지** (톱 메시지, 댓글 모두)

**댓글 (상세):**

- 전체 내용 자유 형식
- 마크다운 → Slack mrkdwn 변환 필수 (Quick Reference 참조)

### Step 3 — 멘션 처리 (선택)

멘션이 필요한 경우:

| 유형               | 조회 방법                                                                               | 포맷                  |
| ------------------ | --------------------------------------------------------------------------------------- | --------------------- |
| 서브팀 (`@handle`) | `slack_search_public_and_private`로 채널 내 과거 멘션 검색, `<!subteam^SXXX>` 형식 추출 | `<!subteam^SXXXXXXX>` |
| 유저 (`@name`)     | `mcp__claude_ai_Slack__slack_search_users`                                              | `<@UXXXXXXX>`         |
| 채널 전체          | —                                                                                       | `<!channel>`          |
| 온라인 멤버        | —                                                                                       | `<!here>`             |

서브팀/유저 ID는 **추측하지 말고 반드시 조회**해서 확인한다.

### Step 4 — 사용자 확인

두 메시지(톱/댓글)를 사용자에게 모두 보여준 뒤 게시 승인을 받는다. 승인 전에는 절대 전송하지 않는다.

### Step 5 — 게시

```
1. slack_send_message(channel_id=<채널>, text=<톱 메시지>)
   → 응답의 ts 저장 (예: "1778134885.750479")

2. slack_send_message(channel_id=<채널>, thread_ts=<톱의 ts>, text=<댓글>)

3. 두 메시지의 ts를 사용자에게 보고
```

## Quick Reference — mrkdwn 변환

| Markdown        | Slack mrkdwn                      |
| --------------- | --------------------------------- |
| `**굵게**`      | `*굵게*`                          |
| `_이탤릭_`      | `_이탤릭_`                        |
| `[텍스트](URL)` | `<URL\|텍스트>`                   |
| `` `코드` ``    | `` `코드` ``                      |
| ` ```블록``` `  | ` ```블록``` `                    |
| `> 인용`        | `> 인용`                          |
| `---` 구분선    | 지원 안 됨 — `──────` 등으로 대체 |

## Common Mistakes

- **마크다운 그대로 전송**: Slack은 `**bold**`를 인식하지 않음. `*bold*`로 변환해야 함
- **사용자 확인 생략**: 자동 전송 금지. 반드시 내용을 보여주고 승인 받기
- **thread_ts 누락**: 댓글 전송 시 thread_ts 없으면 새 독립 스레드가 됨
- **서브팀 ID 추측**: `<!subteam^ABC123>` 형식에서 ID를 임의로 넣으면 멘션이 깨짐. 채널 검색으로 확인
- **채널 ID/이름 혼용**: `slack_send_message`의 channel 파라미터는 ID(`C…`)를 사용
- **이모지 삽입**: 톱 메시지·댓글 어디에도 이모지를 넣지 않는다
- **톱 메시지 산문화**: 한눈에 스캔이 어려움. 기본은 불릿. 산문은 사용자가 명시했을 때만
- **격식체 마무리**: 톱 메시지 마지막 줄은 `상세 내용은 댓글에 작성할게요.` 구어체 1줄. "작성합니다" X
- **익명 화자 라벨 추정**: 음성 트랜스크립트의 `Participants 1/2/3…` 같은 라벨이 보이면 임의로 실명에 매핑하지 말 것. 사용자에게 매핑을 확인하거나 채널 과거 게시물에서 참석자 명단을 조회 후 진행
- **도메인 용어 임의 변환**: "백엔드", "DA" 등 화자가 사용한 용어를 임의로 바꾸지 말 것. 트랜스크립트 원문/사용자 정정 표현을 그대로 유지
- **파일 분할 ≠ 세션 분할 가정 금지**: 녹음 파일이 여러 개여도 자동으로 Part 1/2/3 구분을 만들지 말 것. 파일 간 관계(이어지는 회의인지, 별개 세션인지)가 불명확하면 사용자에게 확인. 사용자가 "이어지는 녹음"이라고 했거나 시간 간격이 없으면 단일 흐름으로 주제별 정리.
