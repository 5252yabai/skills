# 커밋 메시지 구조
    <타입>: <제목>
    <본문>
    <꼬리말>
    
# 작업 순서
1. git diff를 파악한다. $(git diff --cached --diff-algorithm=minimal)
2. 최근 커밋이력을 파악한다. $(git log -3 --pretty=format:"%B" | sed 's/"/\\"/g')
3. git diff와 커밋이력을 토대로, 커밋 메세지를 작성한다.

# 커밋 메시지 규칙
1. 제목은 50자 이내로 작성
2. 제목과 본문 사이는 한 줄 띄움
3. 제목은 명령문으로 작성
4. 본문은 "무엇을", "왜" 변경했는지 설명
5. 본문의 각 행은 72자 이내
6. 꼬리말은 이슈 번호 참조 시에만 사용 (현재 husky를 통해 자동으로 이슈 번호가 추가되므로 직접 입력할 필요가 없어)
7. 타입 종류
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅
- refactor: 코드 리팩토링
- test: 테스트 코드
- chore: 기타 변경사항
8. 한국어로 작성하세요. 단, 파일 이름, 함수 이름은 원문 그대로 사용해도 좋습니다.
9. 간결하게 유지하여 과도한 설명 피하기
10 .불필요한 형식적인 언어 사용 피하기
