## Instructions
1. 현재 브랜치가 어느 브랜치로 부터 분기되었는지 확인합니다.
2. 분기된 브랜치와 현재 브랜치의 file diff를 확인합니다. (git diff)
3. 분기된 브랜치로부터 현재 브랜치까지의 커밋 이력을 확인합니다. (git log)
4. file diff와 커밋 이력을 토대로 PR 메세지를 작성합니다.
5. 작성한 PR 메세지를 클립보드에 복사합니다.

## Best Practices
1. PR 메세지는 마크다운 포멧으로 작성해주세요.
2. PR 메세지에 전체적인 흐름을 파악할 수 있는 흐름도를 포함해주세요.
3. 과장되지 않고 간결하고 명료한 단어를 사용해주세요.
4. 쓸모없는 이모지는 붙이지 말아주세요.
