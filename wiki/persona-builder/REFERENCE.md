# Persona Builder — 상세 방법론

## Slack 수집 요령

**신원 해석**

- `slack_search_users`에 이메일 `jslee01@miridih.com`을 넣어 user id·display name·handle을 얻는다.
- 대체 표기(예전 닉네임)가 있으면 함께 수집 대상에 넣는다.

**발화 수집**

- `slack_search_public_and_private` 쿼리에 `from:<handle>` + 날짜 연산자(`after:YYYY-MM-DD before:YYYY-MM-DD`)를 조합해 증분 창만 긁는다.
- 채널·DM 전부 대상. 결과는 페이지네이션으로 끝까지.

**맥락 확보 (핵심)**
반응 패턴은 "무엇에 대한 반응인가"가 있어야 의미가 있다. 각 발화마다:

- 스레드면 `slack_read_thread`로 부모 메시지 + 앞선 흐름을 가져온다.
- 스레드가 아니면 `slack_read_channel`로 그 timestamp 전후 몇 개를 가져와 맥락을 잡는다.
- → **(맥락 자극 → 내 응답)** 쌍으로 저장. 이 쌍이 반응 패턴·의사결정 방식의 1차 근거다.

**출처 링크**

- 각 근거는 Slack permalink로 인용한다(가능하면). permalink를 못 얻으면 `채널명 / 날짜 / 발화 요지`로 대체 표기.

**부하 관리**

- 최초 전체 빌드는 무겁다. 월 단위로 배치 처리하고, 각 배치가 끝나면 중간 상태를 문서 `## 갱신 상태`에 적어 재개 가능하게 한다.
- 특정 채널/기간을 샘플링하거나 건너뛰었다면 **반드시 커버리지 메모에 남긴다.** "다 봤다"처럼 보이는 조용한 절단 금지.

## 위키(1.Raw) 수집

- 증분 창: 폴더명 `YYYY-MM`, 파일 mtime, 또는 `git log --since=<마지막실행일> --name-only`로 신규·변경 파일 식별.
- `1.Raw/{YYYY-MM}/ai_session/`은 AI 대화 원본 — **사용자 발화 턴만** 추려 근거로 쓴다(AI 답변은 근거 아님, 단 맥락 참고 가능).
- 사용자가 직접 쓴 글(블로그·일지·Confluence 동기화 문서)이 가치 판단·문체의 1급 근거다.

## 4차원 추출 루브릭

각 차원은 상황 공간(업무결정·코드리뷰/기술·대인응답·문서작성) 태그와 함께, **규칙 + 실제 인용 + 출처**로 남긴다.

**1) 반응 패턴 — 트리거 → 반응**

- 반복되는 (자극 유형 → 사용자 반응) 매핑. 예: "모호한 요구가 오면 → 먼저 범위를 좁히는 질문을 던진다."
- 감정 반응(짜증·흥분·냉담)도 트리거와 함께 기록.

**2) 가치 판단 기준**

- 무엇을 중요시/경계하는가, 트레이드오프에서 무엇을 우선하는가.
- "간결함 > 완결성", "속도 vs 정확성" 같은 우선순위 축을 근거 발화로 뒷받침.

**3) 커뮤니케이션 스타일**

- 어투, 문장 길이·구조, 즐겨 쓰는 표현·이모지, 상황별 레지스터(공식/캐주얼) 차이.
- **실제 발화 원문**을 예시로 그대로 인용(문체 복제용).

**4) 의사결정 방식**

- 판단 시 밟는 논리 순서, 요구하는 근거 수준, 리스크·불확실성 다루는 법, 번복 조건.
- 기술 판단(코드리뷰/설계)과 사람 판단(대인)을 구분.

## 충실도 원칙 (있는 그대로)

- 미화·정제 금지. 조급함·편향·감정·반복하는 실수는 **`## 맹점·편향` 섹션에 명시**한다.
- 이유: 대리 판단이 진짜 사용자를 흉내내려면 결함까지 재현해야 정확하다.

## 불확실 처리 (문서에 명시할 지시)

페르소나 문서 상단 사용법에, 적용 에이전트를 향해 다음을 못박는다:

- **불확실해도 인캐릭터로 무조건 추정한다. 보류·되묻기 금지.**
- 근거가 약하면 가장 가까운 규칙·가치 축에서 외삽한다.

## 증분 병합 규칙

기존 페르소나 문서가 있을 때:

- **강화**: 기존 규칙과 같은 방향의 새 근거 → 인용 목록에 추가하고 확신을 높인다.
- **진화**: 새 근거가 기존과 다르면 → 규칙을 갱신하고, `(과거: … → 현재: …)`로 변화를 남긴다. 과거를 지우지 말고 진화로 기록.
- **신규**: 새 패턴 → 새 규칙으로 추가.
- **캘리브레이션**: 사용자가 채점 메모를 남겼으면 반영해 규칙을 교정.
