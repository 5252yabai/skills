# Query

사용자가 위키에 질문할 때 수행할 절차.

## 위키 경로

위키 Vault의 절대 경로: `/Users/leejaeseok/texts_wiki`

모든 파일 읽기·쓰기는 이 경로 기준으로 수행한다. 현재 작업 디렉토리와 무관하다.

## 위키 규칙

- `1.Raw/`는 **절대 수정하지 않는다** (읽기 전용)
- `2.Wiki/`만 관리한다 (entities/, themes/, analyses/)
- `[[문서명]]` 형식 wikilink 사용
- 한국어로 작성, H1(`#`) 금지 — `##`(H2)부터 시작
- Analysis 템플릿: `/Users/leejaeseok/texts_wiki/.claude/skills/wiki-ingest/assets/analysis.md`

## 절차

### 1. 질문 분석 및 유형 분류

질문의 범위와 유형을 파악한다:

- **사실 확인형** ("X에 대해 뭐라고 했어?"): 특정 Entity/Theme 페이지 조회 → 즉답. 탐색 깊이 얕음.
- **시간 비교형** ("2023년과 2024년에 어떻게 변했어?"): Theme 페이지의 `## 시간순 변화` 섹션 우선 참조. 없으면 해당 시기 Raw 폴더(`1.Raw/YYYY-MM/`) 탐색.
- **패턴 발견형** ("반복적으로 등장하는 주제는?"): 여러 Theme/Entity 페이지를 교차 분석. Analysis 생성 후보.
- **교차 분석형** ("개발 메모와 철학 메모의 교차점?"): 서로 다른 영역의 페이지를 비교. Analysis 생성 후보.

### 2. 위키 탐색

3단계 탐색을 수행한다. 이전 단계에서 충분한 답을 얻으면 다음 단계를 생략할 수 있다.

1. **1차: index.md 스캔** — `2.Wiki/index.md`에서 질문과 관련된 페이지명을 식별한다.
2. **2차: Wiki 페이지 읽기** — 관련 Entity/Theme/Analysis 페이지를 읽는다. 각 페이지의 `## 관련 원본` 섹션에서 참조된 원본 목록을 수집한다.
3. **3차: Raw 직접 탐색** — Wiki에서 충분한 답을 얻지 못한 경우에만 수행한다.
   - 파일명 검색: Glob으로 `1.Raw/**/*키워드*` 패턴 매칭
   - 내용 검색: Grep으로 `1.Raw/` 내 키워드 검색
   - 시기 특정 질문: `1.Raw/YYYY-MM/` 폴더를 직접 탐색

탐색 범위가 넓어지면 사용자에게 중간 보고 후 진행 여부를 확인한다.

### 3. 답변 구성

위키 내용을 기반으로 답변한다:

- 핵심 답변을 먼저 제시한다
- 원문 인용 시 `> 인용문` 블록과 `[[1.Raw/문서명]]` 링크를 함께 제공한다
- 시간순 변화가 있으면 연대기 순으로 정리한다
- 위키에서 답을 찾지 못한 부분은 명시적으로 "위키에 관련 기록 없음"으로 표기한다
- 답변 말미에 탐색한 페이지 목록을 간략히 기재한다:
  ```
  📎 참조: [[themes/글쓰기와-사유]], [[entities/에리히-프롬]] 외 N개 원본
  ```

### 4. 기존 페이지 보강

탐색 중 발견한 새 연결을 기존 Wiki 페이지에 반영한다:

- `## 연결` 섹션에 새 wikilink 추가
- `## 미탐구 영역`에서 이번 질문으로 탐구된 항목 갱신
- 누락된 `## 관련 원본` wikilink 추가
- **기존 서술은 변경하지 않는다. 추가만 한다.**

변경이 없으면 이 단계를 건너뛴다.

### 5. 통찰 보존 판단

다음 조건 중 하나 이상에 해당하면 `2.Wiki/analyses/`에 Analysis를 생성한다:

- 기존 위키 페이지에 없던 **교차 연결**이 발견됨 (2개 이상의 Theme/Entity 간)
- **시간에 따른 관점 변화** 패턴이 새롭게 식별됨
- 답변 과정에서 **3개 이상의 원본**을 교차 참조하여 종합한 결과물이 있음
- 사용자가 질문한 관점이 기존 Theme의 `## 미탐구 영역`에 해당함

다음의 경우에는 Analysis를 **생성하지 않는다**:

- 기존 Entity/Theme 페이지의 내용을 그대로 전달한 경우
- 단일 원본의 내용을 요약한 경우
- 사용자가 명시적으로 저장을 원하지 않는 경우

Analysis 생성 시 [analysis 템플릿](/Users/leejaeseok/texts_wiki/.claude/skills/wiki-ingest/assets/analysis.md)을 기반으로 작성한다.

### 6. 인덱스·로그 업데이트

위키에 변경이 있었으면 수행한다:

- 새 페이지 생성 시: `2.Wiki/index.md`에 등록 (가나다순)
- 모든 변경: `2.Wiki/log/YYYY-MM.md`에 기록 (최신이 위)
- 형식: `- Query: "{질문 요약}" → {변경 내용} ([[관련/문서]])`
- 작업유형 대괄호 금지. 문서 경로는 backtick 대신 wikilink 사용. wikilink 문법 충돌 시 클릭 가능한 Markdown 링크 사용.

## 예시 질문

- "내가 '사랑'에 대해 어떤 생각들을 해왔어?" → 사실 확인형
- "독서 노트에서 반복적으로 등장하는 주제는?" → 패턴 발견형
- "2023년과 2024년에 관심사가 어떻게 변했어?" → 시간 비교형
- "개발 관련 메모와 철학 메모의 교차점이 있어?" → 교차 분석형

## 주의사항

- `1.Raw/` 원본은 절대 수정하지 않는다
- 기존 위키 페이지의 서술을 함부로 변경하지 않는다 — 연결·원본 추가만 허용
- Analysis를 남발하지 않는다 — 생성 기준을 엄격히 적용한다
- 사용자의 사색적이고 간결한 문체를 존중한다
