# Wiki Context

현재 작업에 관련된 맥락을 개인 위키에서 탐색하여 제공하는 워크플로우.
wiki-query가 "위키에 질문"이라면, wiki-context는 "위키를 참고하여 현재 작업을 보강"하는 역할이다.

## 위키 경로

절대 경로: `/Users/leejaeseok/texts_wiki`

구조:

- `2.Wiki/index.md` — 전체 카탈로그 (탐색 진입점)
- `2.Wiki/entities/` — 특정 대상 페이지 (인물, 책, 기술, 프로젝트 등)
- `2.Wiki/themes/` — 주제 종합 페이지 (시간순 변화, 관점 발전 포함)
- `2.Wiki/analyses/` — 교차 분석 페이지
- `1.Raw/` — 원본 자료 (블로그, 일지, Confluence 문서, 세션 기록 등)

## 절차

### 1. 맥락 필요 판단

사용자의 현재 작업에서 키워드를 추출한다:

- 코드 문제: 기술명, 패턴명, 모듈명, 에러 유형
- 아이디어: 주제어, 관련 개념
- 의사결정: 기술 선택지, 아키텍처 패턴

### 2. 위키 탐색

`/Users/leejaeseok/texts_wiki`에서 3단계 탐색을 수행한다. 이전 단계에서 충분하면 다음을 생략한다.

1. **index.md 스캔** — `2.Wiki/index.md`에서 키워드와 관련된 페이지명을 식별한다.
2. **Wiki 페이지 읽기** — 관련 Entity/Theme/Analysis를 읽는다. 특히:
   - Entity의 `## 관련 메모 요약` — 과거 경험과 결론
   - Theme의 `## 시간순 변화` — 관점이 어떻게 발전했는지
   - Theme의 `## 미탐구 영역` — 아직 탐구되지 않은 부분
   - Analysis의 `## 발견` — 이전 분석에서 얻은 통찰
3. **Raw 직접 탐색** — Wiki에서 부족하면 `1.Raw/`을 Glob/Grep으로 검색한다.

### 3. 맥락 전달

탐색 결과를 현재 작업에 녹여서 전달한다. 형식:

```
---
📚 위키 맥락:
- {관련 경험/통찰 1줄 요약} (← [[entities/페이지명]])
- {관련 경험/통찰 1줄 요약} (← [[themes/페이지명]])
- ...
---
```

전달 원칙:

- **간결하게**: 맥락은 보조 정보다. 현재 작업의 본론을 압도하지 않는다.
- **연결하여**: 위키 내용을 현재 문제에 어떻게 적용할 수 있는지 한 줄로 연결한다.
- **출처와 함께**: 더 읽고 싶을 때를 위해 위키 페이지 경로를 남긴다.
- **없으면 없다고**: 관련 맥락이 없으면 "위키에 관련 기록 없음"으로 명시하고 넘어간다.

### 4. 본 작업 수행

맥락 전달 후, 사용자의 원래 요청을 수행한다. 위키에서 얻은 맥락을 답변에 자연스럽게 반영한다.

## 위키는 읽기 전용

이 스킬은 위키를 **읽기만** 한다. 위키에 쓰지 않는다. Analysis 생성이나 페이지 업데이트가 필요하면 `/wiki-query`를 사용한다.
